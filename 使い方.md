# 使い方ガイド（Gijiroku プラグイン）

Lark の録画から、要約・タスク・図解つきの**議事録ドキュメントを自動で作る**ツールです。
ターミナル操作は不要。Claude Code に話しかけるだけで使えます。

---

## 1. 導入（最初の一度だけ）

Claude Code の入力欄で、次の2行を実行します：

```
/plugin marketplace add momiji-project/gijiroku-marketplace
/plugin install gijiroku@gijiroku-marketplace
```

## 2. はじめての作成（初回セットアップ）

`/Gijiroku-my` と入力すると、**初回だけ**セットアップの質問が出ます。答えるだけでOK：

1. （`lark-cli` が無ければ）導入してよいか聞かれる → 「はい」
2. **QRコード**が表示される → スマホの Lark アプリで読み取り → 「OK」と入力
3. 議事録を保存したい **Lark フォルダの URL** を貼り付け
   （例）`https://〈あなたの会社〉.larksuite.com/drive/folder/xxxxxxxx`
4. ファイル名のタグ（例 `(議事録)`）を入力。不要なら「なし」

→ 設定が保存され、そのまま最新の録画から議事録が作られます。

## 3. 毎回の使い方

`/Gijiroku-my` と打つだけ。
あなたが所有する**最新の録画（Minutes／妙記）**を自動で取得し、議事録ドキュメントを作って **URL** を返します。

> 2回目以降はセットアップ不要。`lark-cli` 導入済み・ログイン済みなら即作成されます。

## 4. できあがるもの

指定フォルダに Lark ドキュメントが作成されます。中身：

- 📋 概要 ／ ✅ 決定事項 ／ 📌 タスク・アクション
- 📊 図解（内容に応じてフロー図・マインドマップ・表など最大4点）
- 📄 逐字稿（文字起こし原文）

タイトルは「日付 ＋ タグ ＋ 要旨：会議名」の形になります。

## 5. 設定を変えたいとき

- 保存先フォルダやタグを変えたい → Claude に「議事録の保存先（またはタグ）を変えたい」と言えば設定し直します。
- 設定は各PCの `~/.claude/gijiroku-config.json` にローカル保存されています。

## 6. うまくいかないとき（FAQ）

| 症状 | 対処 |
|------|------|
| 「`lark-cli` が無い」と出る | ウィザードが導入を案内します（Node.js / npm が必要） |
| QRが読めない／期限切れ | もう一度 `/Gijiroku-my` を実行すると新しいQRが出ます |
| 「権限がありません」系のエラー | 追加のQR認証が表示されます。読み取ればOK |
| 録画が見つからない | 対象は「**自分が所有者**の最新録画」です。共有された録画から作りたい場合は配布元に相談（共有版の手順あり） |
| コマンドの許可ダイアログが出る | Claude が代わりに操作しています。「許可」を選んでください |
| 保存先を間違えた | 「保存先フォルダを変えて」と Claude に伝えてください |

## 7. プライバシー

- 動作は**あなた自身の Lark アカウント／会社テナント**内で完結します。
- ログイン情報・保存先トークンは各PCにローカル保存。配布リポジトリには一切含まれません。

---

更新は配布元が `version` を上げて配信します（受け手側は自動で更新が届きます）。
